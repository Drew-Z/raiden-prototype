# 人工演示说明

## 推荐演示入口

优先从主菜单选择 `Chapter Run`。

这是当前版本最完整、最适合展示的一条路线，能够连续覆盖：

- 第一关成长与资源建立
- 章节简报的继承感
- 第二关风暴机关与 Boss 压迫
- 章节结尾与总结页的外层包装

## 推荐演示顺序

1. 主菜单
说明当前版本是双关纵版切片展示候选，演示重点是完整短局闭环。
2. 第一关
展示基础移动、自动射击、火力成长、炸弹资源和第一段敌群节奏。
3. 简报页
说明生命、炸弹与火力的继承，以及双关路线不是简单拼接，而是连续流程。
4. 第二关
展示风暴机关、侧向压迫、资源继承后的高压战场和 Boss 警戒节奏。
5. 章节结尾
展示章节判定、阶段总结和通关后外层流程。
6. 章节总结
展示整条路线的收束、结论和下一步建议。

## 演示时建议强调

- 这是一个双关垂直切片，不是单关原型。
- 第一关负责建立成长与资源节奏，第二关负责兑现压迫感和展示价值。
- 火力、炸弹、Boss 窗口和结果页已经形成一条完整的展示链路。
- 当前版本重点是“可展示、可评审、可继续打磨”，不是长流程正式成品。

## 演示时不要跳过

- 简报页里的继承信息
- 第二关的风暴预警与 Boss 过载窗口
- 章节结尾里的判断区块
- 章节总结里的最终收束信息

## 演示后可直接给出的结论

- 当前版本已经具备对外展示和阶段评审价值。
- 继续推进时，优先级应该是人工试玩 QA、打包和最终展示包装，而不是继续扩新系统。
