# 当前进度记录

## 总体阶段判断

- 当前状态：双关可展示原型
- 所处阶段：已完成最小可玩闭环，正在向双关垂直切片早期推进
- 当前完成度判断：已经适合对外展示双关样片，但还没有进入正式内容生产阶段

## 已完成阶段

### 阶段 1：最小可玩闭环

- 主菜单、开始游戏、战斗、结果页完整打通
- 玩家移动、自动射击、受伤、死亡和一局结束流程成立
- 火力升级、炸弹、Boss、HUD 和结果页基础形态完成

### 阶段 2：展示原型增强

- 敌群前中后段节奏重排
- 火力成长提升到更清楚的 Lv5 链路
- 炸弹资源提示和清屏表现增强
- Boss 拥有更强的分段压迫与阶段切换
- 增加暂停、重开、返回菜单等外层流程

### 阶段 3：展示可读性与关底收束强化

- 结果页重组为更像展示版的总结界面，包含战绩卡片、三段摘要和下一步建议
- Boss 击破后增加战场清场总结卡，让战斗收束与结果页之间有明确过渡
- Boss 入场、相位切换、核心暴露和 overdrive 全部接入事件卡提示
- 关键补给、压位段与夹击段也接入事件卡提示，整局节奏提示更加连续
- 高速侧入和双侧夹击波次增加边缘入口预警，进一步强化战场阅读性

### 阶段 4：视听反馈与 Boss 演出强化

- 增加独立 Boss 入场锁定特效和 Boss 击破冲击特效
- 强化命中、爆炸与炸弹波纹层次，提升击中和清屏的体感
- 增加程序生成的占位音乐，并区分战斗、Boss、overdrive、失败和通关收束
- 为 Boss 入场、Boss 击破和失败收束增加电影边栏包装
- 增加低血量与关底高压阶段的持续危险压屏，补足战场氛围层

### 阶段 5：敌机职责扩展与第二关骨架验证

- 增加新的 `screener` 敌机职责，用慢速大颗粒弹形成可读的屏障火力路线
- 主菜单增加 `Stage 01` 与 `Stage 02` 选择入口
- 战斗主控改为通过关卡目录切换数据脚本，不再把单关写死在主流程里
- 新增 `Stage 02 // Storm Front` 关卡骨架，用于验证第二关的节奏、Boss 参数和扩展成本
- 结果页和全局状态加入关卡标识，保证多关卡原型下的结算信息仍然清楚

### 阶段 6：第二关打磨与双关章节串联验证

- `Stage 02` 增加更明确的中前段补给窗口、后段狙击叠压和完整事件提示
- `Stage 02` Boss 改成独立的 `storm` 风格，拥有不同于第一关 Boss 的专属火力节奏与外观
- 主菜单新增 `Chapter Run`，可以直接从 `Stage 01` 接入 `Stage 02`
- 章节模式下会继承生命、炸弹和火力，并在关卡交接时给出明确的阶段过渡提示
- 结果页现在能承接下一关交接与章节总结两种结果，不再只适用于单关结算

### 阶段 7：第二关展示完成度提升

- 为第二关补了新的 `suppressor` 压制扇面敌机职责，用宽扇面火力更明确地逼迫玩家提前站位
- 第二关波次中加入 `suppressor` 与 `pincer`、`anchor`、`screen` 的组合，后段节奏比骨架版更完整
- 章节模式进入第二关时增加了更明确的 `LEG 2` 过渡提示和镜头包装
- 结果页增强了章节交接面板与章节总评面板，让 `Chapter Run` 的结束更像完整样片

### 阶段 8：章节收束增强与第二关敌群深化

- 第二关继续强化了 `suppressor` 波次，把压制扇面正式纳入中后段路线决策
- 章节最终收束增加了 `CHAPTER SECURED` 与 `CHAPTER COMPLETE` 级别的战斗内提示
- 结果页现在会显示章节等级、分关等级和更明确的章节总评
- 双关打完后，结果页的 `Retry` 已调整为真正的 `Retry Chapter`

### 阶段 9：章节演出增强与风暴互动

- 第二关新增 `storm_strike` 风暴打击互动点，用预警线和落雷束逼迫玩家重排路线
- 第二关背景层加入更明确的 `storm` 主题视觉，章节第二段的气质和第一关进一步拉开
- 结果页新增章节路线条、下一段简报和章节尾声文案，章节交接与章节结束更像完整演出
- 单独进入第二关时，现在会使用展示用的中段装载，而不是从 `Lv1 / 2 bomb` 硬起手

### 阶段 10：章节过场场景落地

- 新增独立的 `ChapterBriefing` 场景，作为 `Stage 01` 与 `Stage 02` 之间的真正过场
- `Chapter Run` 流程从“结果页直接跳下一关”升级成“结果页 -> Briefing -> 下一关`
- 过场会展示下一关标题、继承装载、章节时间线和下一段 mission brief
- 自动验证已覆盖 `Stage 01 -> Results -> Briefing -> Stage 02` 的完整双关链路

### 阶段 11：章节镜头化过场与尾声分层

- `ChapterBriefing` 从信息页升级成带电影边栏、扫描线和多区块任务面板的章节过场
- 双关章节通关后新增独立 `ChapterOutro` 场景，负责章节尾声、章节总评和下一步 directive
- 章节模式流程升级成 `Stage 01 -> Results -> ChapterBriefing -> Stage 02 -> Results -> ChapterOutro`
- 自动验证已覆盖完整双关章节链路，确认新增场景不会打断 `--autoplay --chapter`

### 阶段 12：第二关环境联动增强

- 第二关新增独立 `storm_sweep` 环境机关，用横向扫掠补强风暴区间的空间压迫
- 新增 `storm_cross` 事件，让纵向 `storm_strike` 与横向 `storm_sweep` 形成十字封线联动
- `ChapterBriefing` 和 `ChapterOutro` 补入章节卡片，让双关路线在交接和尾声阶段更清楚
- 自动验证已覆盖 `storm_cross` 出现后的 `Stage 02` 单关与 `Chapter Run` 完整链路

### 阶段 13：第二关联动编排与 ending 收束增强

- `storm_cross` 现在支持 `support_wave`，环境机关和支援敌群可以在同一个节奏点里联动压路线

- `ChapterOutro` 新增短暂的 ending 封板，让双关通关后的收束更像章节 ending 而不是直接进总结面板

### 阶段 14：双关 ending 分层

## 当前大阶段结论

- 当前版本已经从单关展示样片进入双关可展示原型，章节串联已经成立
- 当前结构已经不只是能承载多关卡，还能承载中场过场、章节交接和章节总评
- 下一步更值得做的是继续强化章节镜头表现和环境互动，而不是继续堆基础系统

- 第二关已经形成自己的敌群语言，不再只是第一关的数值加压版本
- 风暴打击让第二关开始有自己的环境互动逻辑
- `Chapter Run` 已经有比较明确的开始、交接、终结三段包装
- 结果页不仅给出成绩，还承担了章节总评与复盘职责
- 中场 briefing 和章节 outro 已经从结果页中拆出来，章节流分层更清楚
- 第二关开始具备“敌群压迫 + 环境十字封线”这类更接近正式关卡的联动压力
- 第二关已经开始出现“机关先锁路线，敌群再补刀”的正式关卡节奏
- 双关章节的结尾现在已经分成“ending 镜头”和“debrief 面板”两层
- 第二关 Boss 现在也开始和环境机关联动，关底高潮更接近完整关卡设计
- 当前 build 已经非常接近一个可以明确宣称的“双关垂直切片候选”
- 当前工作已经明显进入“打磨和收口”阶段，而不是继续横向扩系统
- 路线 A 现在已经进入最后一轮节拍和包装精修区间

## 当前仍缺少的内容

- 正式音效和真正可长期使用的背景音乐资源
- 更强的命中停顿、击破停顿和视听同步细节
- 更强的镜头驱动过场和更有仪式感的章节尾声演出
- 第二关节拍虽然已可展示，但还没有像第一关那样被细抠到最佳状态
- 更长链路的章节串联、章节间成长或资源设计
- 第二关环境机关虽然已成立，但还缺更多能与敌群协同的复合设计
- 章节 ending 已经独立成 scene，但还可以继续抬升镜头表现
- 现在已经接近一个很明确的“双关垂直切片判断点”
- 如果继续推进，后续重点应转向“精修和包装”，而不是继续横向堆系统
- 路线 A 已经启动，后续更适合做最后一轮精修和封版判断
- 继续推进的话，更合理的是准备封版评审，而不是继续引入新玩法层

## 下一阶段建议

### 优先级最高

- 继续补真正可记忆的章节镜头节点，而不只是静态 UI 包装
- 继续给 `Stage 02` 补更复杂的环境联动和敌群协同
- 把当前 `ChapterEnding` 继续推进成更完整的最终 ending scene
- 然后正式决定是否进入双关垂直切片阶段

### 进入垂直切片前需要确认

- 是继续把现有双关做成更完整的可展示垂直切片
- 还是在确认节奏后继续扩出第三段或更多章节骨架
