# 最终演示检查清单

## 推荐演示路径

优先从 `Chapter Run` 开始，而不是直接进入单关。

推荐顺序：

1. 主菜单展示当前 build 状态和演示入口
2. 进入 `Chapter Run`
3. 在第一关展示火力成长、炸弹资源和敌群节奏
4. 在简报页展示继承状态
5. 在第二关展示风暴机关、Boss 压迫和终盘窗口
6. 让章节结尾和章节总结完整播放

## 演示重点

- 第一关：成长、资源、节奏建立
- 第二关：环境压迫、敌群组合、Boss 收束
- 外层流程：简报、结尾、总结、重开、返回主菜单
- HUD：生命、火力、炸弹、分数、Boss 警戒层是否清晰

## 演示前检查

- 主菜单默认布局正常，没有文字越界
- 中文文本正常显示，没有乱码或英文残留
- 音效音量层级正常，没有单个提示音过爆
- 顶部 HUD 不遮挡主要战场信息
- 结果页、结尾页、总结页可滚动且按钮可点击

## 演示时不要跳过的内容

- 简报页里的继承信息
- 第二关风暴预警与炸弹窗口
- Boss 过载和终结窗口
- 章节结尾里的 `切片判断`
- 章节总结里的 `路线总结` 和 `最终封装`

## 当前结论

当前 build 已经适合作为双关纵版切片的正式展示候选。
后续优先级应该放在人工试玩校准、发布包装和最终说明，而不是继续扩大量新内容。
