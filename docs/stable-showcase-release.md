# 稳定展示版说明

## 版本定位

当前版本是一个可对外展示的双关纵版射击切片，重点验证：

- 从主菜单到战斗、结果、章节结尾、章节总结的完整闭环
- 第一关负责建立火力成长与炸弹节奏
- 第二关负责展示侧压、风暴机关与 Boss 高压收束
- HUD、音效、结果页与外层流程已经达到展示版可用状态

## 当前完整体验

一局流程如下：

1. 从主菜单进入，选择单关或章节连打
2. 第一关建立基础火力与炸弹资源，并在小 Boss 段落完成第一次节奏收束
3. 章节模式下进入简报页，继承生命、炸弹与火力后转入第二关
4. 第二关通过风暴预警、侧向压迫与资源继承形成更明显的街机展示段
5. Boss 登场后进入警戒 HUD、过载窗口与终结窗口的压迫节奏
6. 战斗结束后进入结果页；章节模式会继续进入结尾页与章节总结页

## 本轮收口内容

- 清理了 `HUD` 的残余覆盖逻辑，资源显示改为单一来源
- 活跃流程里的中文文案又检查并统一了一轮，重点补齐暂停面板、清场提示、结果页详情
- 顶部 HUD 的层级继续收紧，Boss 条、常驻资源、短时提示的分工更清楚
- 火力掉落在高火力和章节继承状态下进一步收紧，避免后段成长太轻易
- 第一关和第二关 Boss 血量与攻击频率都略有上调，Boss 段不再过于松
- 炸弹窗口提示前移一档，更贴近高压时刻的真实可用性

## 最新试玩反馈打磨

- `Lv5` 的主炮伤害、支援火力与射速已经进一步拉开，成长感比之前更明确
- 第一关与第二关 Boss 的后段节奏更紧，过载前后的压迫更早形成
- 高火力掉落与章节继承状态进一步收紧，后段成长依然存在，但不再像之前那样轻松灌满
- 炸弹窗口提示更早出现，更方便玩家在高压前完成资源兑现

## 已验证内容

- Godot `--headless --quit` 启动通过
- 单关自动流程通过
- 双关章节流程通过
- 没有新增脚本解析或编译错误
- 当前已知环境告警仅剩 Windows 根证书仓读取失败，不影响本地运行

## 当前控制说明

- `方向键 / WASD`：移动
- 自动持续射击
- `Space`：炸弹
- `ESC`：暂停
- `R`：重开当前流程

- 旧版遗留脚本仍在仓库中保留，当前展示流程已统一切到 `stage2 / chapter` 线路
